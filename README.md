# 骨杖寂骑人

[**English Version**](README-en.md) | **中文说明**

`骨杖寂骑人`，英文名 `DeVect`，是一个把《杀戮尖塔》故障机器人充能球思路移植到《空洞骑士》的 Mod。

## 简介

你不再直接释放三种原版法术，而是把它们转成围绕小骑士运转的 `黄球`、`白球`，以及一个由下砸法术供能的 `黑球槽` 冰盾。

- 火球系法术生成 `黄球`
- 上吼系法术生成 `白球`
- 下砸系法术生成 `黑球槽`
- 每当小骑士因普通敌伤实际受伤、成功拼刀，或在黑冲的第 `3` 个物理步上完成一次有效极限闪避时，系统都会推进 `1` 回合，并让当前各格触发一次 `被动`
- 当球槽已满时，再次施放任意对应法术，会把最右边的旧球挤出，并触发那一格的 `激发`

这套系统保留了 HK 的施法输入习惯，同时把战斗节奏改造成更偏 Defect 的“养球、轮转、挤出、换盾”玩法。

## 名称含义

中文名 `骨杖寂骑人` 对应这次融合的两个来源：

- `骨杖` 取自小骑士的骨钉意象，也呼应把原法术改写为围绕战斗节奏运转的攻击体系
- `寂骑人` 延续了“故障机器人”的谐音趣味，也贴合小骑士沉默、空洞、容器般的身份

英文名 `DeVect` 则是对 `The Defect` 的变形命名。

- 读感上保留了 `Defect` 的机械感与故障感
- `Vect` 可以理解为借自 `Vessel`，对应《空洞骑士》中小骑士作为“容器”的身份
- 合在一起，既像一个失控的机器人名，也像一个脱离既定命运的容器代号

## 核心规则

- `复仇之魂 / 暗影之魂` 对应 `黄球`
- `嚎叫 / 深渊尖啸` 对应 `白球`
- `下砸 / 黑暗降临` 对应 `黑球槽（冰盾）`
- 每次施法仍然消耗 Soul，`法术扭曲者` 仍会减少法术消耗
- 未获得某一系法术时，不能生成对应球，FSM 也不会被错误拦截
- 黄球和白球的伤害都基于你当前的实际骨钉伤害，而不是原版法术面板
- `力量` / `亡者之怒` 会先提高基础骨钉伤害，因此会间接抬高黄球和白球收益
- 黑球槽不直接造成伤害，被动获得 `1` 片冰花，激发获得 `3` 片冰花
- 每 `4` 片冰花吸收 `1` 点伤害，最多累计 `16` 片冰花，也就是最多吸收 `4` 点伤害
- `IceShieldDisplay` 会把当前冰盾显示在 HUD 上

## 球槽规则

| 规则 | 效果 |
|------|------|
| 默认槽位 | 最多同时持有 `3` 格 |
| 吸虫之巢 | 球槽扩展为 `4` 格 |
| 新球生成 | 总是从左侧插入，旧球整体向右移动 |
| 激发顺序 | 最右边的球最早离场、最先触发激发 |
| 场景切换 | 保留当前球序、储存数值与冰盾 |
| 长椅保存 | 清空已生成的球和当前冰盾 |

## 白法 / 黑法说明

空洞骑士的三种法术各自都有两档：

- `白法`：初始版本
- `黑法`：强化版本

在本 Mod 中，同一种效果会根据对应法术当前是白法还是黑法，读取不同的基础规则。

也就是说：

- 黄球看 `火球` 当前是白法还是黑法
- 白球看 `上吼` 当前是白法还是黑法
- 黑球槽由 `下砸` 法术生成，但当前白法和黑法共用同一套冰盾规则

## 三种槽位说明

### 黄球 (Yellow Orb)

| 项目 | 白法 | 黑法 |
|------|------|------|
| 生成方式 | 火球法术 | 火球法术 |
| 被动 | 对附近随机一个敌人造成约 `1/4` 当前骨钉伤害 | 对附近随机一个敌人造成约 `1/3` 当前骨钉伤害 |
| 激发 | 对附近随机一个敌人造成约 `0.75x` 当前骨钉伤害 | 对附近随机一个敌人造成 `1x` 当前骨钉伤害 |
| 定位 | 稳定单体补刀、持续磨血 | 更强的稳定单体输出 |

### 白球 (White Orb)

| 项目 | 白法 | 黑法 |
|------|------|------|
| 生成方式 | 上吼法术 | 上吼法术 |
| 初始伤害 | 约 `1/4` 当前骨钉伤害 | 约 `1/3` 当前骨钉伤害 |
| 被动 | 对范围内所有敌人造成当前储存伤害，然后自身伤害 `-1` | 对范围内所有敌人造成当前储存伤害，然后自身伤害 `-1` |
| 额外规则 | 当伤害降到 `0` 时会碎掉并消失 | 当伤害降到 `0` 时会碎掉并消失 |
| 激发 | 对范围内所有敌人造成 `当前储存伤害 x2` | 对范围内所有敌人造成 `当前储存伤害 x2` |
| 定位 | 早期较温和的群伤球 | 更强的清场与持续 AoE 压制 |

说明：白球的白法 / 黑法差异主要体现在 `初始伤害`，被动衰减和 `2x` 激发结构不变。

### 黑球槽 (Ice Shield)

| 项目 | 白法 | 黑法 |
|------|------|------|
| 生成方式 | 下砸法术 | 下砸法术 |
| 被动 | 获得 `1` 片冰花 | 获得 `1` 片冰花 |
| 激发 | 获得 `3` 片冰花 | 获得 `3` 片冰花 |
| 防御规则 | 每 `4` 片冰花吸收 `1` 点伤害，最多吸收 `4` 点伤害 | 每 `4` 片冰花吸收 `1` 点伤害，最多吸收 `4` 点伤害 |
| HUD | `IceShieldDisplay` 会显示当前冰盾 | `IceShieldDisplay` 会显示当前冰盾 |
| 定位 | 防守型槽位，用来换取容错与贴身节奏 | 防守型槽位，用来换取容错与贴身节奏 |

说明：黑球槽现在只负责堆冰盾，不再是旧版的储伤黑球。

## 护符联动

| 护符 | 效果 |
|------|------|
| `萨满之石` | 不再强化原版法术，改为强化黄球和白球的伤害结算 |
| `吸虫之巢` | 不再改火球形态，改为让球槽上限从 `3` 变成 `4` |
| `法术扭曲者` | 仍然只影响法术消耗 |
| `灵魂捕手` / `噬魂者` | 当前没有额外改造，保持原本作用 |
| `力量` / `亡者之怒` | 会影响当前骨钉伤害，因此会间接提高黄球和白球的效果 |

### 萨满之石的当前规则

萨满之石不再使用旧版固定 `+2` 加值，而是对当前会结算伤害的球效果改成：

- `额外伤害 = ceil(baseNailDamage * 0.2 * 该结算点倍率)`

其中：

- `baseNailDamage` 指你当前真实骨钉伤害，已经包含 `力量`、`亡者之怒` 等已有修正
- `该结算点倍率` 指这次伤害本来相对骨钉伤害的倍率

例子：

- 黄球 `黑法被动` 的基础倍率是 `1/3`
- 所以萨满额外值是 `ceil(baseNailDamage * 0.2 / 3)`

补充说明：

- 黄球会在 `被动` 和 `激发` 两个结算点分别按各自倍率计算萨满额外值
- 白球只会在 `初始伤害生成` 时吃一次萨满加成，之后被动和 `2x` 激发直接沿用当前储存值
- 黑球槽当前不吃这套加成，因为它只提供冰盾

## 玩法理解

- 黄球最稳定，适合不断穿插火球维持单体输出
- 白球更适合清杂与持续 AoE 压制，尤其适合利用多种回合推进来源滚雪球
- 黑球槽更偏防守，吃的是“稳定推进回合 -> 稳定触发被动或激发 -> 把冰盾攒起来”的节奏
- 如果你只解锁了白法，没有解锁黑法，黄球和白球会有明显的前中后期成长差异，黑球槽当前则不区分两档数值

## 推荐玩法

- `黄球流`：持续平 A + 穿插火球，打稳定单体输出
- `白球流`：靠上吼攒白球，再用受伤、拼刀或极限闪避把回合快速往前推
- `黑球槽流`：靠下砸法术和球序轮转攒冰盾，用来顶风险、换站位、扛贴身战损
- `混合流`：用白球控场、黄球补伤、黑球槽补容错，最像故障机器人的轮转手感

## 提示

- 如果你发现法术没有正常放出来，这通常是预期行为，它们已经被转成球或冰盾槽了
- 如果某个方向的施法完全不生效，先检查你是否已经获得那一系法术
- 黑球槽需要 `4` 片冰花才能吸收 `1` 点伤害，最多存到 `16` 片冰花，也就是最多吸收 `4` 点伤害
- `IceShieldDisplay` 会把当前冰盾显示在 HUD 上，场景切换后也会继续保留
- 在长椅保存会清空已生成的球和当前冰盾，所以不要指望靠存档把它们带进下一次战斗
- 装了 `吸虫之巢` 后，球序管理会更重要，因为第 `4` 格会明显改变激发顺序，也会影响黑球槽多久被挤出

## 构建与安装

1. 把 `local.props.example` 复制成 `local.props`。
2. 在 `local.props` 里把 `GameDir` 设成 `Hollow Knight/hollow_knight_Data/Managed` 目录。
3. 如果 `MMHOOK_Assembly-CSharp.dll`、`MMHOOK_PlayMaker.dll` 或 `MonoMod.RuntimeDetour.dll` 不在 `GameDir` 下，再额外设置 `ApiDir`。不填时会默认沿用 `GameDir`。
4. 如果你想改 post-build 安装位置，再设置 `InstallDir`。不填时默认输出到 `$(GameDir)/Mods/DeVect`。
5. 也可以不用 `local.props`，改成在构建时传 `/p:GameDir=...`、`/p:ApiDir=...`，或设置 `HOLLOW_KNIGHT_MANAGED_DIR`、`HOLLOW_KNIGHT_API_DIR`。
6. 构建 `DeVect.csproj`。构建前会检查 `Assembly-CSharp.dll`、`MMHOOK_Assembly-CSharp.dll`、`MMHOOK_PlayMaker.dll`、`MonoMod.RuntimeDetour.dll` 是否存在。
7. 构建成功后，项目会把 `DeVect.dll`，以及存在时的 `.pdb` / `.xml` 复制到安装目录，并在同目录生成 `DeVect.zip` 和 `SHA256.txt`。
